# QorTroller — Full Project Assessment

**Date:** 2026-06-13
**Author:** Claude Code session (HWFL-1 /goal close-out)

## 1. What it actually is, in one breath

QorTroller is the reference implementation of **V.A.P.I.** (Verifiable Autonomous Physical Intelligence) — a DePIN sub-category where the *physical-input source is also the cryptographic agency-holder over the data it produces*. Concretely: a gamer's controller generates biometric-grade input data, and the gamer cryptographically owns that data. The thesis is that cheating *cannot exist* when humanity is cryptographically proven at the input layer and the player retains data sovereignty — you don't punish cheating, you make it structurally impossible to assert.

## 2. State of development — honest grade

**Engineering maturity: advanced prototype, testnet-only, single-operator.** This is not a toy and not a product. It is a deeply-built research protocol with ~5,766 automated tests, 173 fail-closed CI invariants, and 66 deployed contracts on IoTeX testnet (58 active per the Cycle 15 contract-status audit). But the credibility gaps are real and self-disclosed:

- **No mainnet. No token. No external Solidity audit. N=3 player corpus.** The protocol states this plainly in its own `ARCHITECTURE.md`.
- **The separation-ratio story is mixed and disclosed as such:** the AIT probe clears the tournament gate (ratio 1.199, N=37, all inter-player pairs >1.0), but full-corpus free-form gameplay does *not* separate players (ratio ≈ 0.06 — the known WIF-009 plateau). Identity-grade biometric ID has an EER ceiling (~29%); the protocol pivoted from "identity" to "presence" (the L9 arc) precisely because that is the honest claim it can make today.
- **Concentration risk is named, not hidden:** single Windows machine, single operator wallet, single-copy MFG Root CA (the F-DECON-3.2 / OA-1 finding, surfaced by the DECON-1 streams and re-nagged every HWFL-1 cycle).

**What is genuinely solid:**

- 228-byte PoAC wire format, FROZEN, never modified.
- Five-plus FROZEN-v1 cryptographic chains (GIC cognitive-continuity, WEC operational-continuity, CORPUS-SNAPSHOT, CONSENT, TEMPORAL-BEACON) — each a SHA-256 hash-chain with a frozen domain tag and a formula the project refuses to break.
- Post-quantum layer (Arc 7) via the Decoupled Sidecar Pointer — ML-DSA-65 signatures live off-chain, only 32-byte commitments cross the wire, so the 228-byte floor is preserved.
- GIC_100 milestone reached (100 consecutive clean grind sessions, cryptographically chained).
- A 38-ID operator agent fleet with one agent (Guardian) doing autonomous HSM-rooted signing at 0 IOTX, on-chain anchored.
- The verification-first discipline itself — drift gets caught mechanically. (HWFL-1's Sensor A v0.2 surfaced two stale claims in `CLAUDE.md` this session — the "49 contracts" headline and the bridge test count — that human session-end discipline had missed.)

## 3. Use cases / features / purposes — current (software-only, testnet)

### A. Anti-cheat / humanity attestation (the core)
The 9-level PITL stack (L0 structural → L6 haptic challenge-response) fuses HID presence, behavioral TinyML classification, IMU-gravity discrepancy detection, biometric Mahalanobis fingerprinting, and temporal-rhythm analysis into a `humanity_probability`. Hard-cheat codes block tournament eligibility. **Purpose:** prove a live human on a certified controller, now, at the input layer — not via screen-scraping or kernel anti-cheat.

### B. Data sovereignty / consent
Per-category gamer consent (TOURNAMENT_GATE / ANONYMIZED_RESEARCH / MANUFACTURER_CERT / MARKETPLACE), enforced so the *bridge can never grant or revoke consent on the gamer's behalf* — only the gamer's own wallet can. **Purpose:** the gamer owns the data their hands produce.

### C. Verified-Human Replay (VHR, Arc 5)
A ZK pipeline (Groth16/circom) that sanitizes a gameplay trace (φ-quantization removes biometric detail) and produces a replay proof a buyer can verify *was generated by a proven human* without seeing the raw biometrics. **Purpose:** a marketplace for provably-human gameplay data.

### D. Proof of Session Recency (PoSR, Arc 6)
Binds session open/close to IoTeX block hashes so a replay cannot claim recency it does not have — closes stale-relisting, pre-computation, and tournament-backdating attacks.

### E. Silicon-rooted device identity (Path A)
Manufacturer-authoritative device birth registry + cert chain. Today: software-backed reference CA. The whole HWFL-1 loop (Sensors A/B/C + dev-kit BOM) is the *planning scaffold* for making this real in hardware.

### F. Marketplace + tournament infrastructure
Listings, tier multipliers computed cryptographically, buyer-category ZK verification, operator-agent governance.

## 4. What it becomes when the hardware aspirations land

This is the part the HWFL-1 loop exists to enable. The "QorTroller" is currently a *certified third-party device* (DualSense Edge CFI-ZCP1). The aspiration is a **purpose-built controller** where the protocol's strongest discriminators are native silicon, not inferred from a borrowed device.

### The hardware staircase (Sensor C's Rungs 1→4)

- **Rung 1 (largely LIVE):** registry + CA + cert infrastructure proven in software.
- **Rung 2 (2/7 LIVE — the current frontier):** the dev-kit. ESP32-class MCU + secure element (the ATECC608B/608C-TFLXTLS family the Sensor B narratives tracked — see the Cycle 16 Path A spec amendment) + Hall-effect/TMR sticks + IMU + **adaptive-trigger mechanism** (the single strongest L4 discriminator — Sony-class force-curves at 1 kHz / 8-bit that translator hardware like Cronus/XIM physically cannot synthesize) + capacitive touchpad. Gated on **Stage A measurements** (Empirical Unknown #1: intra-vs-inter-player trigger Mahalanobis > 1.0; Empirical Unknown #4: Hall-effect same-batch stick separability > 20%).
- **Rung 3:** factory provisioning — TrustFLEX pre-provisioned secure elements, MFG CA chained to a hardware HSM root (which also closes the OA-1 / OA-4 single-copy-CA finding permanently), two-supplier discipline, partner-handoff package.
- **Rung 4:** the spec *as* a compliance standard, device-identity registry interop, IIP-64 post-quantum precompile dependency (external, IoTeX core).

### What changes qualitatively when hardware ships

1. **Per-PoAC silicon rooting (Path A v2)** — every 228-byte record, not just VHP renewals, signed in a secure element. Requires solving the ~3 ms-sign vs 1000 Hz-pipeline buffering problem. This is the leap from "proven human session" to "proven human *cognition cycle*."

2. **The adaptive trigger becomes a native challenge-response channel** — the controller *issues* haptic/trigger stimuli and measures reflex-band response (80–280 ms human voluntary-reaction window), making bot-spoofing a biomechanical-synthesis problem, not a data-replay problem.

3. **The one open lever — breadth — gets a clean harvester.** The honest blocker today is N=3 players; identity-grade separation needs population. The BCC (Behavioral Capture Chain) is built dormant to harvest *only* presence-certified causal sessions, so a purpose-built controller in many hands generates a provenance-clean corpus that the borrowed-device path cannot.

4. **L8 Bluetooth co-presence + L4 v2 sensor stack** become measurable on owned silicon (subject to the same-model-separability study that does not exist in public literature yet — the protocol is explicit that cross-session controller-identity claims are out of scope until that study runs).

5. **The category claim closes:** when the input device is purpose-built, the "physical source *is* the cryptographic agency-holder" thesis stops being aspirational architecture and becomes the literal silicon. That is the point at which V.A.P.I. graduates from "reference implementation on a certified third-party device" to "the QorTroller."

## 5. The candid bottom line

QorTroller is a **cryptographically rigorous, intellectually honest, exceptionally well-instrumented protocol prototype** that has done the hard conceptual and software work — and is disciplined enough to mechanically flag its own documentation drift. Its credibility gaps are exactly the ones it names out loud: tiny corpus, testnet-only, single-operator concentration, no external audit, no hardware yet. Its path forward is concrete and staged (the Rung 1→4 staircase), and the gating items are real engineering / measurement milestones, not hand-waving. The single highest-leverage move remains **breadth** — more humans playing on provenance-clean hardware — and every piece of the protocol is built to be ready for that moment without compromising the FROZEN primitives that make the whole thing verifiable.

### Immediate non-hardware actions worth the operator's attention (surfaced by the loop)

- **OA-1** — MFG CA backup (5-min action; operational-fragility-only now, since the path leaked to public history per F-CYCLE9-1).
- **OA-4** — HSM-root migration (closes both the single-point-of-failure pattern *and* the disclosure trail; the only move that does both).
- **F-HWFL-5-2** — *CLOSED Cycle 16* (the canonical Path A spec no longer points partners at the NRND ATECC608A; now specifies the active 608B/608C-class family with mandatory polling-based timing).
