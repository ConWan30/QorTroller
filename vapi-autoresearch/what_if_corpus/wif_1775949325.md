### INVERSION Classification Rubric (Phase 193 — FleetSignalCoherenceAgent)

When ANALYSIS encounters an INVERSION finding from FleetSignalCoherenceAgent, classify
it before acting. The 3 current INVERSION rules (COMMITMENT_PREDATES_CONSENT,
BADGE_WITHOUT_RENEWAL_PARENT, RULING_PREDATES_CALIBRATION) all detect causal chain
violations via Provenance DAG walk, but not all DAG timestamp anomalies are attacks.

**Classification Step — Query the DAG delta:**
```sql
SELECT child.node_type, parent.node_type,
       child.created_at as child_ts, parent.created_at as parent_ts,
       (CAST(parent.created_at AS REAL) - CAST(child.created_at AS REAL)) as delta_s
FROM data_provenance_dag child
JOIN data_provenance_dag parent ON child.parent_node_id = parent.node_id
WHERE delta_s > 0   -- child timestamp precedes parent = inversion
ORDER BY delta_s DESC LIMIT 10;
```
Examine `delta_s` (seconds by which child precedes parent):

**SEMANTIC INVERSION** (investigate as potential violation):
- `delta_s > 3600` (>1 hour gap): child was registered before parent existed — impossible
  under correct sequencing. Root causes: (a) clock skew on two separate hosts, (b)
  retroactive DAG node insertion by DataCuratorAgent Task 1, (c) adversarial timestamp
  manipulation.
- Affects rule: `COMMITMENT_PREDATES_CONSENT` when delta_s > 3600 on COMMITMENT_HASH node.
- Action: STOP tournament prep; audit DataCuratorAgent Task 1 `register_calibration_session()`
  call site; verify NTP sync on bridge host.

**TEMPORAL INVERSION** (expected at N < 30, ratio < 1.0 — do NOT pause tournament prep):
- `delta_s <= 300` (<=5 minutes): Inversion within a single pipeline execution window.
  Root cause: corpus growth cycles — new sessions are ingested, LOO centroid
  recalculation fires (BP-001 decay re-weights sessions), and the DAG registers the
  new calibration node slightly after the ruling node that depended on the prior
  centroid. This is structurally expected when N < 30 (corpus still growing).
- Affects rule: `RULING_PREDATES_CALIBRATION` when ratio < 1.0 (Phase 143 observed).
- Action: Log and monitor. `delta_s <= 300` temporal inversions at ratio < 1.0 are
  evidence of a healthy growing corpus, not an attack. Resolve via
  `resolve_coherence_entry()` (Tool #147) with note "TEMPORAL_EXPECTED_N<30".
- Phase 193 state: at N=29, ratio=0.998 — RULING_PREDATES_CALIBRATION inversions
  are TEMPORAL (expected). Do NOT treat as security events.

**STRUCTURAL INVERSION** (data registration bug, not an attack):
- `delta_s` between 300–3600 (5 min–1 hour): Inversion larger than a pipeline window
  but smaller than clock skew threshold. Root cause: a DataCuratorAgent Task 1 failure
  mid-run left a partially-registered DAG node; on retry, the child node was re-inserted
  with a newer timestamp while the parent node retained the original timestamp.
- Affects rule: `BADGE_WITHOUT_RENEWAL_PARENT` when parent_node_id is NULL.
- Action: Verify the DataCuratorAgent Task 1 idempotency guard
  (`INSERT OR IGNORE` on node_id). Re-run DataCuratorAgent provenance cycle.
  Not a security event but indicates DataCuratorAgent pipeline health issue.

**Decision matrix (operator fast-path):**
| delta_s | Classification | Pause tournament prep? | Action |
|---------|---------------|----------------------|--------|
| > 3600s | SEMANTIC | YES | Audit + escalate |
| 300–3600s | STRUCTURAL | NO (advisory) | Fix DataCuratorAgent pipeline |
| <= 300s AND ratio < 1.0 | TEMPORAL | NO | Log, resolve as EXPECTED |
| <= 300s AND ratio >= 1.0 | SEMANTIC | YES | Audit — state change expected to eliminate these |

**NEVER suppress an INVERSION finding by calling `resolve_coherence_entry()` before
checking `delta_s`. INVERSION findings at ratio=0.998 with `delta_s <= 300` are
TEMPORAL (corpus growth artifact) — they require logging, not escalation.**
