Proposal: Formally specify Class K adversary model + Phase 99B+/Phase 158 detection architecture:
  - Class K adversary uses synthetic EDA generator intercepting BLE GSR packet stream (48B magic
    0x47535201 format) and injecting biologically plausible SCR signals (Butterworth-filtered,
    baseline drift, peak-like structure at biologically valid inter-peak intervals).
  - Gap: MockGSRGrip (scipy Butterworth + Gaussian SCR) and a sophisticated Class K synthetic EDA
    generator produce signal morphologies that are indistinguishable at the feature-extraction level
    (sympathetic_arousal_index, gsr_game_event_correlation, baseline_conductance_drift,
    cognitive_load_variance). Feature-level detection alone cannot close Class K.
  - Mitigation path: certLevel=2 hardware certification in VAPIHardwareCertRegistry (Phase 99A) must
    require HMAC-SHA256 challenge-response at the BLE packet level. ESP32-S3 device private key signs
    each GSR batch with a bridge-issued nonce. Synthetic EDA generators cannot forge HMAC without
    device private key.

  ESP32-S3 grip and bridge; injects pre-recorded or synthetically generated EDA signals with plausible

  Implication: sympathetic_arousal_index is in normal human range (0.2–0.8); gsr_game_event_correlation
  appears positive because injected SCR peaks are time-shifted to coincide with known game event latency
  windows; baseline_conductance_drift shows plausible session-fatigue slope; cognitive_load_variance
  produces valid inter-SCR interval variation. L7 advisory code 0x33 GSR_CORRELATION_ABSENT never
  fires because correlation IS present — but with synthetic data, not physiological response. The
  current MockGSRGrip(seed=42) design inadvertently defines the Class K attack surface: an adversary
  with access to the MockGSRGrip source can replicate its signal characteristics exactly.
  Cryptographic grounding: BLE advertisement is unauthenticated; current 48-byte GSR packet format
  (magic 0x47535201 + arousal_millis + correlation_millis + ts_ns) contains no HMAC field. A synthetic
  EDA generator with packet injection capability (BLE advertisement spoofing) is economically motivated
  by VHP mint access — tournament entry + prize eligibility. Injecting plausible GSR signals requires
  only a $15 ESP32-S3 and the open MockGSRGrip source as a reference implementation.
  Mitigation: Phase 99B+ Class K Anti-Spoofing: extend GSR packet format to 80 bytes (48B original
  + 32B HMAC-SHA256). ESP32-S3 firmware generates per-packet HMAC using device private key seeded
  from ATECC608A secure element. certLevel=2 in VAPIHardwareCertRegistry stores HMAC public key.
  Bridge validates HMAC on each GSR packet before L7 feature extraction; rejects unsigned packets
  with 0x33 GSR_CORRELATION_ABSENT advisory at WARNING severity. Class K adversary cannot forge
  HMAC without physical access to ATECC608A key material.
