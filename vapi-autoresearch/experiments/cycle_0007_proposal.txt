## W1-028: P1 Temporal Persona Break — One-Way Ratchet on Separation Ratio
The downward trend N=11→1.261, N=14→0.789, N=20→0.569 is not a data shortage artifact.
It is a structural failure: P1 intra-player Mahalanobis variance is growing as sessions from
different calendar weeks are included, driving the centroid toward inter-player space.
Each new P1 session that clusters differently from prior P1 sessions expands the intra-player
covariance, narrowing the Mahalanobis gap to P2/P3. At the limit (intra ≈ inter), ratio → 0
and the touchpad separation gate fails permanently regardless of N.
Mitigation: session-date clustering analysis (W2-028) — compute P1 centroid per calendar week;
if inter-week distance > intra-week variance, mask old sessions and re-enroll from latest persona.
See TOURNAMENT BLOCKER: separation ratio 0.362 (pooled), 0.569 (touchpad_corners N=20).

## W2-028: Session-Date Clustering — Persona-Windowed Calibration Recovery
When P1 inter-week centroid distance > intra-week variance, P1 has multiple biometric personas.
Detection algorithm: group P1 sessions by ISO week; compute mean feature vector per week;
compute pairwise Euclidean distance between weekly centroids; if max_inter_week_distance
> 1.5 × mean_intra_week_std, flag PERSONA_BREAK. Mitigation: retain only the most recent
persona window (most recent contiguous cluster). Re-run analyze_interperson_separation.py
with --persona-window flag. Expected recovery: ratio above 0.70 defensibility gate when
persona-pruned corpus retains ≥ 5 sessions per player from the stable recent window.
This connects to Phase 111 AdjudicationRegistry.sol and ioSwarm quorum — the VHP mint gate
must not authorize a device whose separation ratio was computed from a persona-broken corpus.
Phase candidate: Phase 169 — analyze_interperson_separation.py +--persona-window flag +

W1 — P1 Temporal Non-Stationarity Creates One-Way Ratchet on Separation Ratio
  Failure mechanism: P1 intra-player variance grows monotonically as sessions from different
  calendar weeks are included in the enrollment corpus. Each new week adds sessions that cluster
  differently (grip posture shift, hardware wear, physiological variation), expanding the
  intra-player covariance ellipsoid outward toward inter-player space. The Mahalanobis distance
  formula (ratio = inter_mean / intra_mean) converges to 1.0 and then below 1.0 as intra
  approaches inter. This is a structural failure mode, not a data shortage problem: adding more
  sessions makes the ratio WORSE because each session carries the current persona's feature
  distribution, which may contradict prior sessions. Observed empirically:
  N=11 → ratio=1.261, N=14 → ratio=0.789, N=20 → ratio=0.569. The trend is non-random —
  it is the mathematical signature of a persona break. Cryptographic grounding: the
  SeparationRatioRegistry.sol commitment (Phase 153) is SHA-256(ratio_str + N + players_sorted +
  ts_ns). A commitment made at N=11 (ratio=1.261) is already inconsistent with the live
  state at N=20 (ratio=0.569), creating an on-chain/off-chain inconsistency that is legally
  discoverable during tournament dispute resolution. The current separation ratio of 0.569 is
  below the configurable defensibility gate (min_separation_ratio=0.70), making this a
  TOURNAMENT BLOCKER. The dry_run=True enforcement state means no live blocks are issued yet,
  but tournament eligibility cannot be granted under these conditions.
  Implication: without persona-break detection and persona-windowed re-enrollment, the ratio
  will continue declining toward 0.40 and then 0.30 as more P1 sessions are captured. The
  inter-player separation that exists for P2 vs P3 (~1.3) is masked by P1's expanding variance,
  making the entire 3-player corpus appear non-separable. The W1-002 calibration deadline
  failure mode transitions from a data-availability problem to a data-quality problem that
  cannot be solved by capturing more sessions of the same type.
  Mitigation: session-date clustering analysis (W2-028). Group P1 sessions by ISO calendar week.
  Compute pairwise Euclidean distance between weekly centroids. If max inter-week distance

  flag applies this pruning before ratio computation, recovering a defensible corpus from the
  stable recent persona. Phase 169 candidate.

W2 — Session-Date Clustering Detects Persona Breaks and Recovers Defensible Calibration
  Mechanism: Extend analyze_interperson_separation.py with _detect_persona_break(player_id):
  1. Group sessions by ISO week (session_ts.isocalendar().week)
  2. Compute mean feature vector (centroid) per week: week_centroids[week] = mean(sessions)
  3. Compute pairwise Euclidean distance between all week-centroid pairs
  4. Compute intra-week std: mean(std_per_week across all weeks)
  5. If max(pairwise_distances) > 1.5 × mean_intra_week_std: PERSONA_BREAK detected
  6. Retain only the most recent contiguous calendar cluster (latest N weeks without break)
  7. Re-run ratio analysis on persona-pruned corpus
  The --persona-window CLI flag gates this behavior. persona_break_log table records:
  player_id, break_ts, n_sessions_pruned, ratio_before, ratio_after, persona_window_start.
  GET /agent/persona-break-status returns: persona_break_detected, n_pruned, ratio_recovery,
  persona_window_start, timestamp.
  Phase candidate: Phase 169 (~3h effort) — persona detection + pruning pipeline + bridge API +
  analyze_interperson_separation.py extension. Connects to ioSwarm quorum: the ioSwarm
  adjudication coordinator (Phase 131 IoSwarmAdjudicationCoordinator) should treat persona-broken
  separation ratios as equivalent to ratio < 1.0 when issuing VHP mint authorization via the
  Phase 110 VHP gate. This requires a new PERSONA_BREAK signal fed into the quorum task spec.
  Mechanism exclusive to VAPI: requires Phase 142 diagonal auto-fallback + Phase 143 proper LOO
  + Phase 150 defensibility gate + Phase 153 SeparationRatioRegistry commitment + Phase 157
  FleetConsensusSnapshotAgent — no competing gaming anti-cheat protocol has this composable
  calibration infrastructure. The persona-break detection transforms a raw biometric failure
  mode into a cryptographically auditable, on-chain-provable quality gate.
